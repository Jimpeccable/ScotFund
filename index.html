Database & Persistence Roadmap

Comprehensive requirements for transitioning to a full-service cloud architecture.

1. Infrastructure Options

Recommended: Supabase (PostgreSQL)

Integrated Auth, Realtime DB, and Edge Functions. Perfect for rapid scaling and low latency in Scotland.

Alternative: MongoDB Atlas

Flexible document-based storage. Ideal if fund data formats become highly varied or unstructured.

2. Data Schema Design
  • Users: uuid, email, pwd_hash, role (User/Admin/Super), profile_data_json, last_login
  • Opportunities: uuid, title, funder_id, url, amt_min, amt_max, deadline, sectors[], region, quality_score
  • Shortlists: uuid, user_id, opp_id, status, notes, tasks_json, date_saved
  • Audit Log: uuid, action_type, user_id, timestamp, details_json
3. Security & Auth Requirements
JWT-based Session Management
Row-Level Security (RLS) policies
SSL Encryption for all API calls
Rate limiting for the Scanning Engine
Migration Strategy

Transitioning from localStorage requires a 3-phase approach: 1. Deploy Cloud DB, 2. Scripted data migration via JSON export, 3. Update 'saveData' and 'getFromStorage' to utilize asynchronous API fetch calls.
